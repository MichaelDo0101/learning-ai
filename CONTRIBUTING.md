# Đóng góp cho Task AI Wiki / Contributing to Task AI Wiki

Cảm ơn bạn quan tâm đóng góp! / Thanks for your interest in contributing!

**Task AI Wiki** là dự án giáo dục mở do [Task AI](https://taskai.vn) vận hành, phái sinh từ [datawhalechina/easy-vibe](https://github.com/datawhalechina/easy-vibe) (mã nguồn mở), phát hành theo giấy phép [CC BY-NC-SA 4.0](./LICENSE).

## 🇻🇳 Tiếng Việt

### Cách đóng góp

- **Báo lỗi / góp ý nội dung:** mở [Issue](https://github.com/MichaelDo0101/learning-ai/issues). Ghi rõ trang/bài và mô tả vấn đề.
- **Thảo luận, đặt câu hỏi:** dùng [Discussions](https://github.com/MichaelDo0101/learning-ai/discussions).
- **Sửa nội dung / thêm bài:** fork repo → tạo branch → gửi Pull Request.

### Quy trình Pull Request

1. Fork và clone repo.
2. Cài đặt: `npm install`.
3. Chạy thử local: `npm run dev` (mở `http://localhost:5173`).
4. Sửa nội dung trong `docs/` (tiếng Việt ở `docs/vi-vn/`, tiếng Anh ở `docs/en/`).
5. Build kiểm tra: `npm run build` phải thành công.
6. Commit, push, mở Pull Request mô tả rõ thay đổi.

### Lưu ý nội dung

- Markdown được biên dịch như Vue — tránh ký tự `<` trần và `{{` trong văn xuôi.
- Đặt mọi lệnh/code/prompt trong khối code có rào (fenced code block).
- Giữ giọng văn thực chiến, có ví dụ và dẫn nguồn khi có thể.

## 🇬🇧 English

### How to contribute

- **Report bugs / content feedback:** open an [Issue](https://github.com/MichaelDo0101/learning-ai/issues) with the page/lesson and a clear description.
- **Discuss, ask questions:** use [Discussions](https://github.com/MichaelDo0101/learning-ai/discussions).
- **Edit content / add lessons:** fork → branch → Pull Request.

### Pull Request workflow

1. Fork and clone the repo.
2. Install: `npm install`.
3. Run locally: `npm run dev` (open `http://localhost:5173`).
4. Edit content under `docs/` (`docs/vi-vn/` for Vietnamese, `docs/en/` for English).
5. Verify the build: `npm run build` must succeed.
6. Commit, push, and open a Pull Request describing your change.

### Content notes

- Markdown is compiled as Vue — avoid bare `<` and `{{` in prose.
- Put every command/code/prompt in a fenced code block.
- Keep a practical tone with examples and sources where possible.
